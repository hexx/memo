# 課題解決プロンプト: メモ一覧の検索入力にデバウンスを追加

## 1. 役割定義 (Persona)
あなたはフロントエンドのパフォーマンス最適化に詳しいソフトウェアエンジニアです。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: `src/routes/index.tsx` の検索入力がキー入力のたびに API リクエストを発行しています。メモ数が増えるとサーバー負荷と無駄なネットワークリクエストが発生します。
- **再現手順**: 検索ボックスに文字を入力する。DevTools のネットワークタブで毎回 `/api/memos?q=...` のリクエストが発生していることを確認。
- **関連ファイル/コード**: `src/routes/index.tsx` の `search` state と `useQuery` の依存関係。

## 3. 解決すべきゴール (Goal & Objective)
- [ ] 検索テキストの変更から API リクエスト発行までに 300ms のデバウンスを追加する
- [ ] `useEffect` + `setTimeout` で `debouncedSearch` state を導入する
- [ ] `useQuery` の `queryKey` と `queryFn` を `debouncedSearch` に依存させる
- [ ] 既存の動作（ラベルフィルター、ピン留め表示など）を壊さないこと

## 4. 期待する成果物 (Output Format)
- 修正済みの `src/routes/index.tsx`
- 動作確認の手順

## 5. 実行開始の合図
このプロンプトを理解したら、「タスクを開始します」と宣言し、直ちに関連ファイルの解析から始めてください。
