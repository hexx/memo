# 課題解決プロンプト: メモ編集画面で箇条書きが視覚的に反映されない（prose-org スタイル欠如）

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: メモ編集画面（TipTap ベースの WYSIWYG エディタ）で、ツールバーの「箇条書き」ボタン（または「番号付きリスト」ボタン）を押しても、編集画面上の本文に箇条書きの見た目（黒丸・番号・インデント）が反映されません。TipTap のドキュメント側には `bulletList` / `orderedList` が正しく適用され、ボタンの active 状態（ハイライト）も切り替わるため、「機能しているが描画されない」状態です。結果として本文が通常の段落と見た目で変わらず、ユーザーからは「箇条書きを選んでも反映されない」ように見えます。
- **再現手順**:
  1. `npm run dev:web` で開発サーバを起動し、メモ編集画面を開く。
  2. 本文エリアに適当な文字を入力する。
  3. ツールバーの「箇条書き」アイコン（List）をクリックする。
  4. 行頭に黒丸などの箇条書きマーカーが表示されず、インデントも付かない（平文と同じ見た目になる）。
- **関連ファイル/コード**:
  - `src/components/MemoEditor.tsx`
    - 約 60〜70 行目の `useEditor` 内 `editorProps.attributes.class` に `"... prose-org"` を指定しているが、**`prose-org` というクラスのスタイル定義がプロジェクト内のどの CSS にも存在しません**。
    - ツールバーの箇条書きボタンは `editor.chain().focus().toggleBulletList().run()` を呼んでおり、コマンド自体は正常に働いています（ボタンの `btn(editor.isActive("bulletList"))` ハイライトも切り替わる）。
  - `src/index.css`
    - `@tailwind base`（Preflight）により `ul, ol` が `list-style: none; margin: 0; padding: 0` にリセットされるため、箇条書きが黒丸なし・インデントなしで描画されます。`prose-org` による再定義がないため、このリセットが上書きされません。
    - `@tailwindcss/typography` プラグインは導入されていないため、`prose` / `prose-org` クラスは単なる無意味な文字列になっています。
  - `src/lib/org-serialize.ts`
    - `orgToHtml()`（約 269 行目）は箇条書きを標準的な `<ul><li>`、番号付きリストを `<ol><li>` に出力しています（テスト `src/lib/__tests__/org-serialize.test.ts` でも確認）。これらは正しい HTML ですが、描画用 CSS がないため見た目に出ません。

## 3. 解決すべきゴール (Goal & Objective)
- [ ] `src/index.css` 等で `.prose-org` クラスに対して箇条書き（`ul` / `ol` / `li`）のスタイルを定義し、エディタ上で黒丸・番号・インデントが表示されるようにする。
- [ ] 入れ子リスト（ネストされた `<ul>` / `<li>`）も正しくインデントとマーカーが表示されること。
- [ ] 必要に応じて、見出し（`h1`〜`h3`）、引用（`blockquote`）、コードブロック（`pre` / `code`）、太字（`strong`）、イタリック（`em`）、取り消し線（`s`）、リンク（`a`）などの基本スタイルも `.prose-org` 内に追加し、WYSIWYG としての見た目を整える。
- [ ] 既存のテスト（`src/lib/__tests__/org-serialize.test.ts` 等）を壊さないこと。この修正は**表示（CSS）のみ**とし、`orgToHtml` の出力形式（HTML 文字列）は変更しない。
- [ ] 修正後、`npm run dev:web` で実際に箇条書き・番号付きリストが視覚的に反映されることを確認する（スクリーンショット等で証明しても可）。

## 4. 期待する成果物 (Output Format)
- `.prose-org` のスタイル追加を含む CSS 修正（差分または修正済みファイル）。
- 修正理由の簡潔な解説：Tailwind Preflight が `ul`/`ol` を `list-style:none` にリセットしており、かつエディタが参照している `prose-org` クラスにスタイル定義がなかったことが根本原因。
- 動作確認の結果（開発サーバでの目視、またはビルド成功のログ）。

## 6. 解決記録 (Resolution)
- **実施日**: 2026-07-19 (JST)
- **修正内容**: `src/index.css` の末尾に `.prose-org` のスタイル定義を追加。
  - `ul` → `list-style: disc` + `padding-left: 1.5rem`（箇条書きの黒丸とインデント）
  - `ol` → `list-style: decimal` + `padding-left: 1.5rem`（番号付きリスト）
  - 入れ子リスト: `li > ul` → `circle`、`li > ol` → `lower-alpha`
  - 見出し(h1–h3)・引用(blockquote)・コードブロック(pre/code)・リンク(a) の基本スタイルも追加
- **根拠**: `.prose-org ul` の詳細度 (0,1,1) が Tailwind Preflight の `ul` リセット (0,0,1) を上回るため、エディタ内のリストにマーカーとインデントが表示される。
- **検証**:
  - `npm run build` 成功。コンパイル後の CSS に `.prose-org ul{list-style:disc;...}` 等が出力されていることを確認。
  - `npm test` 成功（5 ファイル / 76 件）。CSS のみの変更のため `orgToHtml` の出力形式は変更せず、既存テストを維持。
- **未実施**: ブラウザでの目視確認（本環境では不可）。`npm run dev:web` で編集画面を開き、箇条書きボタン押下時に黒丸・インデントが表示されることを最終確認してください。
