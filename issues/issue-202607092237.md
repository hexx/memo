# 課題解決プロンプト: org-mode直接編集のWYSIWYG化（Notion/Google Docs風エディタ）

## 1. 役割定義 (Persona)
あなたは高度なソフトウェアエンジニアおよび問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な修正/実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**:
  現在、メモの本文は `src/components/MemoEditor.tsx` 内の `<Textarea>`（プレーンテキストエリア）で直接編集されており、ユーザーは org-mode 記法（`*見出し*`、`- リスト`、`*太字*`、`/イタリック/` など）を手書きする必要がある。これは非エンジニアにとって負担が大きく、Notion や Google ドキュメントのような「見た目通りに編集できる（WYSIWYG）」体験が求められている。
- **再現手順**:
  1. メモ作成/編集ダイアログを開く
  2. 本文エリア（`placeholder="本文（org記法）"`）がプレーンテキストのままである
  3. 見出しや太字を入れるには org 記号を手入力しなければならない
- **関連ファイル/コード**:
  - `src/components/MemoEditor.tsx` — 現在 `<Textarea>` で body を直接編集（`value={body}` / `onChange={setBody}`）
  - `src/lib/api.ts` — `Memo` 型 / `createMemo` / 更新 API（body は org テキスト文字列）
  - `src/server/routes/memos.ts` — メモ CRUD（body を生 org テキストとして保存）
  - `docs/adr/0002-raw-org-text-storage.md` — 本文は「生の org テキスト」で保存するという設計方針（AST 往復によるフォーマット崩れを防ぐため）。**この保存方式は維持する前提**。
  - 技術スタック: React (TanStack Router + TanStack Query) + shadcn/ui + Tailwind、Org パースは `org-toolkit` (hexx/org-toolkit)
  - ターゲット環境: シングルユーザー向け **PWA** として、PC（キーボード・マウス）とスマホ（タッチ・ソフトウェアキーボード）の両方で利用されることを想定

## 3. 解決すべきゴール (Goal & Objective)
- [ ] 本文エリアをプレーンテキストの直接編集から、Notion/Google Docs ライクな WYSIWYG（リッチテキスト）編集 UI に置き換える
- [ ] 見出し、リスト（順序あり/なし）、太字、イタリック、取り消し線、コードブロック、リンクをツールバーやショートカットから挿入・編集できること
- [ ] WYSIWYG エディタの内部モデル → org テキストへのシリアライズを実装し、保存時は既存通り「生の org テキスト」として API に送信すること（ADR 0002 の方針を維持）
- [ ] 既存メモ（org テキスト）をエディタ起動時に org → 内部モデルへデシリアライズし、既存データが崩れず読み込めること
- [ ] 利用可能であれば `org-toolkit` の AST/変換機能を活用し、サポート対象外の org 記法が含まれる場合の挙動（保持/フォールバック）を明確にする
- [ ] 新規エディタ導入にあたり既存のテスト（`src/lib/__tests__/`, `src/server/routes/__tests__/`）を壊さないこと。UI 変更に対するテストが実現可能なら追加する
- [ ] **PC とスマホの入力を両立すること**:
  - PC ではショートカットキー（例: `Ctrl/Cmd+B` で太字、`Ctrl/Cmd+I` でイタリック、見出しレベル切り替えなど）で書式を操作できること
  - スマホ（タッチデバイス）では、ソフトウェアキーボードでのテキスト入力と、ツールバーのタップ操作で書式を挿入・編集できること（ショートカット依存にならないこと）
  - タッチ操作時の操作性（誤タップ防止、ツールバーの配置/折り返し、IME 入力との干渉回避）に配慮すること
- [ ] 制約: シングルユーザー向け PWA / Cloudflare Workers 環境であるため、導入ライブラリは軽量でブラウザバンドルに適していること

## 4. 期待する成果物 (Output Format)
- WYSIWYG エディタへ置き換えた `src/components/MemoEditor.tsx`（および必要に応じて新規コンポーネント）
- org テキスト ↔ エディタ内部モデル の変換ロジック（新規 `src/lib/org-serialize.ts` 等の設計で可）
- 既存の保存/読み込みフロー（API クライアント・サーバ）は org テキストの入出力のまま変更しないこと（互換性維持）
- 修正理由とアプローチの簡潔な解説（どのエディタライブラリを選定したか、org 往復の方針、既知の制限事項）
- 動作確認またはテストの実行結果

## 6. 実装メモ（完了報告）

- **採用エディタ**: TipTap v3（`@tiptap/react` + `@tiptap/starter-kit`）。StarterKit に Heading/リスト/Bold/Italic/Strike/Underline/Code/Link/CodeBlock が含まれるため軽量。
- **新規ファイル**: `src/lib/org-serialize.ts` で org テキスト ⇄ TipTap(JSON) を往復。
  - 読み込み: `orgToHtml()` — Node 組み込みに依存しない自前パーサー（org-toolkit の `index` は `fs`/`path` を import してブラウザバンドルで失敗するため、クライアントでは使用せずサーバー側のみ維持）。
  - 保存: `docToOrg()` — TipTap JSON → 生 org テキスト。ADR 0002 の「生 org 保存」方針を維持（API/サーバは変更なし）。
- **UI**: `src/components/MemoEditor.tsx` を `<Textarea>` から TipTap `EditorContent` + ツールバーに置換。
- **PC/スマホ両立**:
  - PC: TipTap 標準ショートカット（太字 `Mod-B`、イタリック `Mod-I`、下線 `Mod-U`、取り消し線 `Mod-Shift-S`、リスト `Mod-Shift-8/9`、見出し `Mod-Alt-1..3`、コード `Mod-Alt-C`）。
  - スマホ: タップ操作のツールバー（見出しレベル select、太字/イタリック/下線/取り消し線/リンク/リスト/引用/コード）。ショートカットに依存しない。
- **テスト**: `src/lib/__tests__/org-serialize.test.ts` を追加（往復ロジック 9 件）。既存テストを含む全 63 件が通過。`npm run build` も成功。
- **既知の制限**: テーブル・footnote・プロパティドロワー等の org 要素は編集モデルに持ち込まず、そのままでは WYSIWYG 編集対象外（保存時に再付与はしない）。`#+TITLE:`/`#+FILETAGS:` はメタデータとしてエディタからは除外（タイトル欄・ラベル欄で管理、エクスポート時に再付与）。
