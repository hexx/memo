# 課題解決プロンプト: メモ本文からAIでタイトル自動生成

## 1. 役割定義 (Persona)
あなたは Cloudflare Workers（Hono）と React に精通したフルスタックエンジニアであり、OpenAI 互換の LLM API 連携に習熟した問題解決のエキスパートです。以下のコンテキストとゴールを基に、正確かつ効率的な実装を行ってください。

## 2. 背景・前提条件 (Context)
- **現状発生している問題**: 現在、メモ作成・編集時にユーザーがタイトルを自分で手動入力する必要があります。`src/server/routes/memos.ts` の `POST /api/memos` および `PUT /api/memos/:id` は `title` を必須としており、空の場合は `400 { error: "Title is required" }` を返します。ユーザーの要望は「タイトルを自分で書くのではなく、本文（org-mode 記法）の内容から AI で自動生成したい」というものです。
- **LLM 接続方式**: Vercel AI SDK（`ai` の `generateText` と `@ai-sdk/openai-compatible` の `createOpenAICompatible`）を利用し、OpenAI 互換の任意のエンドポイント（OpenCode Go 等）を呼び出します。接続情報は以下の 3 つの環境変数から取得します（いずれも未設定の場合は既存の手動入力フローへフォールバックすること）。
  - `AI_API_KEY`: API キー（Authorization: Bearer に使用）。**シークレット**
  - `AI_BASE_URL`: ベース URL（例: `https://opencode.ai/zen/go/v1`）
  - `AI_MODEL`: 使用するモデル名（例: `qwen3.7-max`、未設定時は既定値）
- **再現手順**: メモ作成画面で本文のみを入力し、タイトルを空のまま保存しようとした場合、現在はエラーになります。これを「本文から AI 生成したタイトル」で保存できるようにする。
- **関連ファイル/コード**:
  - `src/server/routes/memos.ts`（POST `/api/memos`、PUT `/api/memos/:id` — タイトル必須バリデーション）
  - `src/server/db/schema.ts`（`memos` テーブルの `title` カラム）
  - `src/lib/api.ts`（フロントエンド API クライアント）
  - `src/components/MemoEditor.tsx`（メモ編集 UI）
  - `src/routes/memos/$memoId.tsx`、`src/routes/index.tsx`（作成・編集画面）
  - `wrangler.toml`（Cloudflare Workers の環境変数 / シークレット設定）

## 3. 解決すべきゴール (Goal & Objective)
- [ ] タイトル自動生成用のサーバーサイド関数を実装する（Vercel AI SDK の `generateText` と `@ai-sdk/openai-compatible` の `createOpenAICompatible` を利用し、本文から短い日本語タイトルを生成する）。`src/server/lib/generateTitle.ts` に切り出す。
- [ ] `AI_API_KEY` / `AI_BASE_URL` / `AI_MODEL` の 3 つを Cloudflare Workers の環境変数（開発時は `.dev.vars`、本番は `wrangler secret put`）として読み込めるように `wrangler.toml` を更新する。
- [ ] `POST /api/memos` および `PUT /api/memos/:id` で `title` が空（または未指定）の場合は、本文から AI 生成したタイトルを採用する。手動入力された `title` がある場合は従来通りそれを優先する。
- [ ] 3 つの環境変数のいずれかが未設定の場合、あるいは LLM 呼び出しが失敗/タイムアウトした場合は、例外を投げるのではなく既存の手動入力フローへ安全にフォールバックする。
- [ ] フロントエンド（`MemoEditor` 等）に「本文からタイトルを AI 生成」ボタン、または本文入力時の自動生成オプションを追加し、生成されたタイトルをタイトル欄に反映する。
- [ ] 既存のテスト（`src/server/routes/__tests__/memos.test.ts` 等）を壊さず、タイトル生成処理にはモック API を用いた単体テストを追加する。

## 4. 期待する成果物 (Output Format)
- 修正済みの `src/server/routes/memos.ts`（および必要に応じた新規 util ファイル）
- `wrangler.toml` の環境変数定義更新と、`.dev.vars` の例・README への環境変数説明の追記
- フロントエンドの UI 変更（`MemoEditor` 等への生成ボタン追加）
- タイトル生成の単体テスト（モック API を利用）
- 修正理由とアプローチの簡潔な解説
- 動作確認またはテストの実行結果（既存 `vitest` の通過を含む）
